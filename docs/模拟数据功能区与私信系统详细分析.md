## 一、模拟数据功能区架构

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        前端应用层                                 │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────────┐ │
│  │  Components │  │   Screens   │  │    React Navigation     │ │
│  └─────────────┘  └─────────────┘  └─────────────────────────┘ │
│                              │                                   │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                      Hooks 层                               ││
│  │  useContacts()  useChatHistory()  useSendMessage()        ││
│  └─────────────────────────────────────────────────────────────┘│
│                              │                                   │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                   Service 层                                ││
│  │  messageService  forumService  userService  partnerService ││
│  │         ▲                                    ▲             ││
│  │         │                                    │             ││
│  │    USE_MOCK = false                    USE_MOCK = false  ││
│  │         │                                    │             ││
│  └─────────┼────────────────────────────────────┼─────────────┘│
└────────────┼────────────────────────────────────┼───────────────┘
             │                                    │
    ┌────────┴────────┐               ┌─────────┴─────────┐
    │   Mock 数据     │               │    API Client    │
    │ src/data/mock/  │               │  (真实后端 API)   │
    └─────────────────┘               └───────────────────┘
```

| `src/data/mock/me.ts` | 个人中心 | 当前用户动态、收藏、点赞 |

## 三、模拟数据启用方式

### 3.1 服务层开关

每个服务文件顶部有一个 `USE_MOCK` 常量：

```typescript
// 文件: src/api/services/message.service.ts
const USE_MOCK = false;  // 改为 true 启用模拟数据

export const messageService = {
  async getContacts(): Promise<Contact[]> {
    if (USE_MOCK) {  // 当为 true 时返回模拟数据
      const { mockContacts } = await import('../../data/mock/messages');
      return mockContacts;
    }
    // 当为 false 时调用真实API
    const { data } = await apiClient.get(ENDPOINTS.MESSAGE.CONVERSATIONS);
    return (Array.isArray(data) ? data : []).map(...);
  },
};
```

## 四、私信系统详解

### 4.1 页面结构

```
MessagesScreen (消息列表页)
├── 顶部通知快捷入口
│   ├── 点赞通知 (NotifyLikes)
│   ├── 关注通知 (NotifyFollowers)
│   └── 评论通知 (NotifyComments)
├── 搜索栏（可展开）
├── 联系人列表
│   └── ContactRow (每个联系人)
└── 长按操作菜单 (Action Sheet)
    ├── 标记已读/未读
    ├── 置顶/取消置顶
    ├── 静音/取消静音
    └── 拉黑

ChatScreen (聊天详情页)
├── 顶部导航栏
├── 消息列表 (FlatList)
│   ├── DateSeparator (日期分隔线)
│   └── ChatBubble (消息气泡)
│       ├── 文本消息
│       └── 功能卡片 (Function Card)
└── 输入栏
    ├── 文本输入框
    ├── 发送按钮
    ├── 相机按钮
    ├── 麦克风按钮（录音）
    └── 图片按钮
```

### 4.2 消息列表页 (MessagesScreen) 详细功能

#### 通知快捷入口
- **点赞通知**: 心形图标，红色渐变背景，显示未读点赞数
- **关注通知**: 用户图标，蓝色渐变背景，显示未读关注数
- **评论通知**: 评论图标，绿色渐变背景，显示未读评论数
- 点击可跳转到对应通知页面

#### 联系人排序规则
1. 搜索模式下：按原始顺序，不区分置顶
2. 正常模式：置顶联系人 > 普通联系人

#### 长按操作
| 操作 | 效果 |
|------|------|
| 标记已读/未读 | 切换消息已读状态 |
| 置顶/取消置顶 | 将联系人固定在列表顶部 |

### 5.2 时间格式化函数

```typescript
// message.service.ts 中的时间格式化

// 联系人列表时间格式
function formatMessageTime(iso: string): string {
  const d = new Date(iso);
  const now = new Date();
  const diff = now.getTime() - d.getTime();
  const mins = Math.floor(diff / 60000);
  const hours = Math.floor(diff / 3600000);
  const days = Math.floor(diff / 86400000);

  if (mins < 1) return '剛剛';
  if (mins < 60) return `${mins}分鐘前`;
  if (hours < 24) return `${hours}小時前`;
  if (days === 1) return '昨日';
  if (days < 7) return `${days}日前`;
  return `${d.getMonth() + 1}/${d.getDate()}`;
}

// 聊天记录时间格式
function formatChatTime(iso: string): string {
  const d = new Date(iso);
  return `${d.getHours().toString().padStart(2, '0')}:${d.getMinutes().toString().padStart(2, '0')}`;
  // 返回格式: "18:35"
}
```

## 七、开发注意事项

### 7.1 时间处理
- 模拟数据使用相对时间（"2小時前"）
- 真实API返回ISO时间戳，需要格式化
- 使用 `formatMessageTime()` 和 `formatChatTime()` 转换

### 7.2 头像处理
- 模拟数据使用首字母（"陳"）
- 真实API可能返回URL或null
- `Avatar` 组件需要处理这两种情况

### 7.3 分页处理
- 当前模拟数据为静态数组
- 真实API需要处理分页参数 `PaginationParams`

### 7.4 匿名处理
- 论坛帖子和评论支持匿名
- 使用 `generateAnonymousIdentity()` 生成一致的匿名身份
